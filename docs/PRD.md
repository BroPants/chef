# Chef 微信小程序 - 产品需求文档 (PRD)

> 版本：v1.0  
> 更新日期：2025-03-04

---

## 1. 产品概述

| 项目 | 说明 |
|------|------|
| 产品名称 | Chef |
| 产品类型 | 微信小程序 |
| 产品定位 | 通过拍摄已做好的菜品，识别菜品并展示所需食材与制作方法 |
| 目标用户 | 想学做菜、想复刻某道菜、对食材用量不熟悉的用户 |

---

## 2. 核心功能需求

### 2.1 菜品识别（P0）

| 需求ID | 需求描述 | 验收标准 |
|--------|----------|----------|
| R-001 | 拍照识别 | 支持小程序内调用相机拍照 |
| R-002 | 相册选择 | 支持从相册选择已有照片 |
| R-003 | 识别结果 | 返回菜品名称及置信度 |
| R-004 | 识别失败处理 | 无法识别时给予明确提示，引导重拍或更换图片 |
| R-005 | 多菜品识别 | （P1）可识别图片中多道菜并分别返回 |

### 2.2 菜谱信息展示（P0）

| 需求ID | 需求描述 | 验收标准 |
|--------|----------|----------|
| R-006 | 食材列表 | 展示所需食材及用量（如「土豆 2 个」） |
| R-007 | 制作步骤 | 分步骤展示做法，支持图文说明 |
| R-008 | 预估时长 | （P1）展示预估制作时间 |
| R-009 | 难度等级 | （P1）展示难度（简单/中等/困难） |
| R-010 | 营养价值 | （P2）展示营养成分、热量等 |

### 2.3 用户功能

| 需求ID | 需求描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| R-011 | 微信登录 | P0 | 支持微信一键登录 |
| R-012 | 收藏菜谱 | P1 | 可收藏识别过的菜谱，支持取消收藏 |
| R-013 | 历史记录 | P1 | 可查看过往识别记录列表 |
| R-014 | 分享功能 | P2 | 可分享菜谱至好友/朋友圈 |

## 3. 用户流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  打开小程序   │ ──► │ 拍照/选图    │ ──► │  上传识别     │ ──► │  展示菜谱    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                            │                    │
                            │                    └──► 识别失败 ──► 提示重试
                            └──► 授权相机/相册
```

**步骤说明：**

1. **启动**：进入首页，突出拍照/选图入口  
2. **拍摄/选图**：调用相机或相册选择图片  
3. **上传**：将图片上传至后端  
4. **识别**：后端调用 AI 接口识别菜品  
5. **展示**：展示菜品名称、食材、制作步骤  
6. **扩展**：收藏、分享、历史记录等

## 4. 技术需求

### 4.1 前端（微信小程序）

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | 原生小程序 / Taro / uni-app | 根据团队情况选择 |
| 相机 | wx.chooseMedia / wx.chooseImage | 获取图片 |
| 上传 | wx.uploadFile | 图片上传 |
| 授权 | 相机、相册、用户信息 | 需处理拒绝授权场景 |

### 4.2 后端

| 模块 | 说明 |
|------|------|
| 接口服务 | 提供图片上传、识别、菜谱查询等 API |
| 图片存储 | OSS/COS 等云存储 |
| AI 识别 | 腾讯云 / 百度 / 阿里等视觉 AI 接口 |
| 菜谱数据 | 自建数据库或第三方菜谱 API |
| 用户系统 | 微信 openid/unionid 管理 |

### 4.3 菜谱数据来源

- 自建菜谱数据库  
- 第三方菜谱 API  
- 大模型生成菜谱（如 GPT、文心等）  

---

## 5. 非功能性需求

### 5.1 性能

- 图片上传前压缩，单张建议 < 500KB  
- 识别结果返回时间 < 5 秒  
- 列表、详情页无明显卡顿  

### 5.2 体验

- 首次使用有简单引导  
- 识别中展示加载态  
- 识别失败有明确文案和操作指引  

### 5.3 安全与合规

- 图片仅用于识别，不长期留存（或符合隐私政策）  
- 遵守微信小程序平台规范  
- 接口使用 HTTPS  

### 5.4 兼容性

- 覆盖主流机型  
- 建议最低支持微信 7.0+  

---

## 6. 风险与依赖

| 类型 | 描述 |
|------|------|
| 识别准确率 | 依赖 AI 能力，需测试与调优 |
| 菜谱覆盖度 | 需保证常见菜品有对应菜谱 |
| 成本 | AI 调用、存储有费用，需做用量与成本评估 |
| 审核 | 小程序需符合微信审核要求 |
