## Phase 1：MVP（最小可行产品）

**目标**：完成核心识别流程，可 demo 演示

| 序号 | 任务 | 对应需求 | 产出 |
|------|------|----------|------|
| 1.1 | 项目初始化 | - | 小程序框架、后端框架、仓库结构 |
| 1.2 | 微信登录 | R-011 | 登录流程、openid 获取 |
| 1.3 | 拍照/选图 | R-001, R-002 | 相机调用、相册选择、图片压缩 |
| 1.4 | 图片上传 | - | 上传接口、OSS 存储 |
| 1.5 | AI 菜品识别 | R-001~R-004 | 对接视觉 AI、返回菜品名 |
| 1.6 | 菜谱数据对接 | - | 菜谱库/API 对接 |
| 1.7 | 菜谱展示页 | R-006, R-007 | 食材列表、制作步骤 UI |
| 1.8 | 错误与空态 | R-004 | 识别失败、网络异常提示 |
| 1.9 | 联调与测试 | - | 端到端流程跑通 |
| 1.10 | 提审上线 | - | 完成微信审核、正式发布 |

**里程碑**：用户可拍照识别菜品并查看菜谱

---

## Phase 2：用户增强

**目标**：提升留存与使用频率

| 序号 | 任务 | 对应需求 | 产出 |
|------|------|----------|------|
| 2.1 | 收藏功能 | R-012 | 收藏/取消、收藏列表 |
| 2.2 | 历史记录 | R-013 | 识别历史列表、清除记录 |
| 2.3 | 预估时长 | R-008 | 菜谱展示时长字段 |
| 2.4 | 难度等级 | R-009 | 菜谱展示难度字段 |
| 2.5 | 多菜品识别 | R-005 | 多结果展示与选择 |
| 2.6 | 性能优化 | 非功能 | 加载速度、缓存策略 |

**里程碑**：用户可收藏菜谱、查看历史，信息更完整

---

## Phase 3：体验升级

**目标**：社交传播与深度使用

| 序号 | 任务 | 对应需求 | 产出 |
|------|------|----------|------|
| 3.1 | 分享功能 | R-014 | 分享至好友、生成分享卡片 |
| 3.2 | 营养价值 | R-010 | 营养成分、热量展示 |
| 3.3 | 菜谱搜索 | 扩展 | 文字搜索菜谱 |
| 3.4 | 使用引导 | 非功能 | 首次使用引导、空态引导 |
| 3.5 | 数据统计 | 扩展 | 埋点、基础数据分析 |

**里程碑**：具备分享与搜索能力，体验更完善
